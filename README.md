# Hi — I build tiny, local-first tools for Claude Code 🛠️

Each one solves **one real daily problem**, with zero dependencies, no network calls, and tests.

| Tool | What it does |
|---|---|
| **[usage-guard](https://github.com/eltonylfgi-blip/claude-code-usage-guard)** | Warns you in-session **before** you hit your real Claude Code plan limits — with an even-pace readout: *should you slow down or push?* |
| **[session-triage](https://github.com/eltonylfgi-blip/claude-session-triage)** | Finds every idle Claude Code session across your projects and shows what each was **still waiting on**. One panel. |
| **[usage-pacer](https://github.com/eltonylfgi-blip/claude-usage-pacer)** | A single-file local web app to pace your weekly Claude quota so it lasts until the reset. |
| **[tikslide](https://github.com/eltonylfgi-blip/tikslide)** | Reads a TikTok photo slideshow (carousel) as clean per-slide Markdown — the part yt-dlp can't do. |

Same philosophy everywhere: **one real problem · tiny footprint · nothing leaves your machine.**

📓 I also keep a public engineering notebook — [cuaderno-madre](https://github.com/eltonylfgi-blip/cuaderno-madre) — real lessons (failures included) from building a file-based personal automation system with AI.
